lmctl

Orchestrate teams of AI coding agents — across providers.

lmctl is a local-first CLI for running teams of AI agents. You define a team in a plain-text teamfile — a Lead plus members, each backed by whichever provider you choose (Claude Code, Codex, Gemini, Qwen, OpenCode, Antigravity) — then route work between them and keep a human in the loop on the steps that matter.

It's not an IDE and not another chatbot. It coordinates the agent CLIs you already run.

Read the docs → already shipped — install and use it today 
npm install -g @lmctl-ai/lmctl

Define a team, then put it to work

# team.lmctl — a plain-text team of agents
_MEMBER_ alias=Lead      provider=claude    # orchestrates, talks to you
_MEMBER_ alias=Coder     provider=codex     # implements
_MEMBER_ alias=Reviewer1 provider=codex     # reviews
_MEMBER_ alias=Reviewer2 provider=claude    # cross-provider review
_MEMBER_ alias=Reviewer3 provider=agy
lmctl plan .            # generate a starter team for a repo
lmctl seed team.lmctl   # spin up the agents
lmctl chat ...          # hand work to the Lead

How work flows

            you
             |
        Lead  (claude)
      .------+----------------.
    Coder              Reviewers
   (codex)       (codex / claude / agy)
      '------------+----------'
              human approval
                   |
               execution

The Lead delegates, members work in parallel across providers, reviewers check each other's output, and you approve the steps that matter. Teams can also connect to other teams (cross-team edges) for larger pipelines.

Why lmctl

  • Provider-agnostic. One control plane over Claude, Codex, Gemini, Copilot, OpenCode, Qwen, and Antigravity — mix per agent, no lock-in.
  • Cross-provider review. Have one provider review another's work; in practice that beats a single model reviewing itself.
  • Durable sessions. Each agent keeps a persistent durable-memory, so long projects stay coherent instead of leaning on session compaction.
  • Cost-aware routing. Put top-tier models on architecture and design, leaner focused models on coding and routine roles.

Get started

npm install -g @lmctl-ai/lmctl
lmctl init            # detect + auth your provider CLIs
lmctl plan .          # generate a starter team

Docs & tutorials: lmctl.com/lmctl. Local-first — it runs on your machine against the agent CLIs you already have; a browser control surface is optional.
