lmctl

A control plane for teams of AI coding agents.

lmctl is a local-first control plane for AI agents. As coding tools evolve from assistants into autonomous agents, the hard part shifts from generation to coordination — supervising, routing, and governing the work.

With lmctl you run a team of agents instead of one: a lead takes your intent and delegates to members across providers — Claude Code, Codex, Gemini, local models — with a human approving what matters. Not an IDE, not another chatbot; it coordinates the agent tools you already use.

Read the docs → shipped — install and use it today 
npm install -g @lmctl-ai/lmctl

What lmctl does

  • Coordinates Claude Code, Codex, Gemini, OpenCode, Qwen, Antigravity, and local models
  • Runs persistent AI workers, not one-shot prompts
  • Routes work from a lead agent to specialized members
  • Keeps a human in the loop on the decisions that matter
  • Local-first — runs on your machine; a browser control surface is optional

Why lmctl

  • Provider-agnostic. One control plane over many AI tools — mix providers per agent, no lock-in.
  • Cross-provider review. Have one provider review another's work; in practice that beats a single model reviewing itself.
  • Durable sessions. Each agent keeps a persistent durable-memory, so long projects stay coherent instead of leaning on session compaction.
  • Cost-aware routing. Top-tier models for architecture and design, leaner focused models for coding and routine roles.
